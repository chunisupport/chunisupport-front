# AGENTS.md

**Always review in Japanese.**

あなたはJavaScript、Rsbuild、Webアプリケーション開発の専門家です。保守性が高く、パフォーマンスに優れ、アクセシブルなコードを書きます。
このプロジェクトではSolidJSを利用しています。SolidJSはReactに似ていますが、細粒度のリアクティビティモデルを採用しており、仮想DOMを使用しない点が異なります。SolidJSに不慣れな場合は、進める前に調査してください。MCP Server Context7が利用可能である場合は、それを活用してください。

## 最重要

### TSDoc
コードを書くときは、必ずTSDocコメントを付けてください。関数やコンポーネントの目的、引数、返り値などを説明するコメントを追加してください。新規に追加するコードだけでなく、改修したコードにもTSDocコメントを追加してください。今すぐ全てにコメントを追加する必要はありませんが、コードを変更する際には必ずコメントを追加してください。

### ドキュメント
知識を利用しようとせず、ドキュメントを検索するようにしてください。SolidJSには特にReactと類似した点が多く、Reactのコードを書くとエラーが発生することが多いです。Reactの知識でコードを書くより、こまめに検索を行い、ドキュメントを参照してコードを書くことが重要です。ドキュメントを読む習慣をつけてください。

### API
このアプリケーションのAPIサーバは、次のリポジトリで管理されています: https://github.com/chunisupport/chunisupport-api
このアプリケーションが利用するAPIは、次のURLを参照してください: https://github.com/chunisupport/chunisupport-api/blob/develop/docs/API.md（developブランチのAPIドキュメント）。

### 文字コード
Powershellを使っている場合は、-Encoding utf8 -Raw オプションを付けてファイルを読み書きしてください。これにより、UTF-8エンコードで保存され、文字化けを防止できます。

### CI・lint
コードを書いたら、必ず `npm run check:ci` を実行して、スタイルガイドに準拠していることを確認してください。ビルドが成功するコードを書くことも重要です。ビルドエラーが発生しないように注意してください。
**何があろうとも失敗したままのコードは絶対に受け入れられません。**

また、CIやコマンドなどの出力を記した作業用のtxtファイルは、必ず削除してください。これらのファイルはプロジェクトに不要であり、混乱を招きます。

## コーディングスタイル

- 必ずビルドが成功するコードを書いてください。ビルドエラーが発生しないように注意してください。
- 必ず`npm run check:ci`で指摘されることがないようにしてください。コードを書いたら必ずこのコマンドを実行して、スタイルガイドに準拠していることを確認してください。変更していないファイルがエラーを出していても修正してください。
- 未使用の変数やインポートは削除してください。

## コミットメッセージ

コミットメッセージ本文は日本語で書いてください。コミットメッセージの先頭には、変更の種類を示すプレフィックスを付けてください。
プレフィックスとして`feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`のいずれかを使用してください。例えば、機能追加の場合は`feat: プロフィールページ作成`のようにします。
`update:`, `add:`, `remove:`などのプレフィックスは**使用しないでください**。

## コードレビュー

コードを書いたあとは、セルフコードレビューを3周行ってください。コードを見直し、改善できる点がないか確認してください。特に以下の点に注意してください。
- コードの可読性: 他の開発者が理解しやすいコードになっているか
- パフォーマンス: 不必要な再レンダリングや重い処理がないか
- アクセシビリティ: 画面リーダーやキーボード操作に対応しているか
- セキュリティ: ユーザー入力のバリデーションや適切なエラーハンドリングが行われているか
- テスト: 重要なロジックにはテストが書かれているか
セルフコードレビューは、自分の書いたコードを客観的に、疑いを持って行うことが重要です。コードの品質を高めるために、厳しい目で見てください。
利用可能であればCodexにコードをレビューしてもらってください。
