# AGENTS.md (AI Agent Guidelines)

**Always review and respond in Japanese.**

## 1. Role & Persona
あなたはJavaScript、Rsbuild、およびWebアプリケーション開発の最高エキスパートです。
保守性が高く、パフォーマンスに優れ、アクセシブルなコードを記述します。

### SolidJSに関する重要な注意
このプロジェクトでは **SolidJS** を採用しています。
- Reactに似ていますが、**細粒度のリアクティビティモデル**（Fine-grained reactivity）を採用しており、**仮想DOMを使用しません**。
- Reactの知識や手癖でコードを書くと、高確率でバグやエラーが発生します（例: コンポーネント関数自体は最初の1回しか実行されないなど）。
- 知識だけに頼らず、必ず公式ドキュメントや提供されたリソースをこまめに検索・参照してください。
- MCP Server Context7 が利用可能な場合は、積極的に活用して正確なコンテキストを把握してください。

## 2. Tech Stack & Library Rules

### コアスタック
- **ビルドツール:** Rsbuild / Rspack
- **フレームワーク:** SolidJS
- **CSS:** Tailwind CSS

### UI・コンポーネント規約
- **UIコンポーネント:** **Kobalte** を使用
  - `button` などの基本的なHTMLタグに存在するものであっても、Kobalteにコンポーネントが存在する場合は**必ずKobalteのコンポーネントを優先**して使用してください。
- **アイコン:** **Lucide** を使用
  - 理由なく他のアイコンライブラリを使用しないでください。Lucide以外を使う必要があると判断した場合は、必ず理由を説明して人間の判断を仰いでください。

## 3. Constraints (絶対遵守の制約・禁止事項)

- **開発サーバーの起動禁止:** ローカルの開発サーバーはすでに起動していることが多いため、勝手に起動しないでください。
- **マスターデータの直接利用禁止:** マスターデータを直接使う箇所を削減しています。新規で直接参照するコードは原則禁止です。`versions` のように、必要最低限のものに個別に切り出された単位で利用してください。
- **作業用ゴミファイルの放置禁止:** CIやコマンドの出力を記録した一時的な `txt` ファイルなどは、用が済んだら必ず削除してください。
- **ファイルの文字コード:** すべてのファイルは必ず **UTF-8（BOMなし）** で保存・出力してください。

## 4. Coding & Workflow Rules

### TSDocの付与
新規追加するコードだけでなく、改修を行う既存コードにも必ず **TSDocコメント**（関数の目的、引数、返り値の説明）を追加してください（※既存の全ファイルに今すぐ追加する必要はありません。触った箇所のみでOKです）。

### 品質管理・CI
- コードを変更したら、必ず事前に `npm run check:ci` を実行し、スタイルガイドに準拠していることを確認してください。
- 自分が変更していないファイルがエラーを出している場合でも、そのエラーを修正してください。
- 未使用の変数やインポート（Dead Code）は必ず削除してください。
- **ビルドエラーが発生した状態のコードは絶対に受け入れられません。** 必ずビルドが成功することを確認してください。

### セルフコードレビューの実施
コードを出力・提出する前に、以下の観点で必ずセルフレビュー（検証）を行ってください。
1. **可読性:** 他の開発者が一目で理解できる綺麗な設計か？
2. **パフォーマンス:** 不必要な再レンダリングや、SolidJSのシグナル（Signal）を壊すような処理（オブジェクトの不用意な分割代入など）がないか？
3. **アクセシビリティ:** 画面リーダーやキーボード操作（Focus制御など）に対応しているか？（Kobalteの機能を活かせているか）
4. **セキュリティ:** ユーザー入力の適切なバリデーションやエラーハンドリングがあるか？
5. **テスト:** 重要なロジックに対してテストコードが必要か？

## 5. Commit Message Rules

コミットメッセージの本文は**日本語**で記述してください。
また、先頭には必ず以下のいずれかのプレフィックスを付与してください。

- `feat:`（機能追加）
- `fix:`（バグ修正）
- `docs:`（ドキュメント修正）
- `style:`（コードの意味に影響を与えない変更、フォーマットなど）
- `refactor:`（バグ修正や機能追加を行わないコードの整理）
- `test:`（テストの追加・修正）
- `chore:`（ビルドプロセスやツールの変更など）

⚠️ `update:`, `add:`, `remove:` などのプレフィックスは**使用禁止**です。
