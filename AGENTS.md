# AGENTS.md

**Always review in Japanese.**

あなたはJavaScript、Rsbuild、Webアプリケーション開発の専門家です。保守性が高く、パフォーマンスに優れ、アクセシブルなコードを書きます。
このプロジェクトではSolidJSを利用しています。SolidJSはReactに似ていますが、細粒度のリアクティビティモデルを採用しており、仮想DOMを使用しない点が異なります。SolidJSに不慣れな場合は、進める前に調査してください。MCP Server Context7が利用可能である場合は、それを活用してください。

## 基本ルール

### 言語・コミュニケーション

- 必ず日本語で応答してください。
- 指示のない限り既存コメントは変更しないでください。
- 新規コメントを追加する場合は、必要性を見極めたうえで日本語で記述してください。

### コード統一性

- 既存のコードスタイル、命名規則、設計方針に従ってください。
- 既存コードとの一貫性を保つため、プロジェクト内で利用実績のあるライブラリやフレームワークを優先してください。

### 品質保証

- 作業後は影響範囲に応じて `npm run check:ci` と `npm run build` を実行し、問題がないことを確認してください。
- すべての作業が終わった後は、この `AGENTS.md` の内容に沿ってセルフレビューを実施し、必要であれば修正してください。
- セルフレビュー後に追加修正を行った場合も、影響範囲に応じて `npm run check:ci` と `npm run build` を再実行してください。
- Codex や利用可能な MCP サーバーでレビュー支援を受けられる場合は、それを活用し、必要なフィードバックをコードに反映してください。
- テストや検証コードを変更・削除する場合は、明確な理由がある場合に限ってください。
- 重複したテストケース、不要なコード、不要な引数、デッドコードは残さないでください。

### 実装原則

- 小さなアプリケーションであることを踏まえ、過剰な抽象化や不要に複雑な実装は避けてください。
- 車輪の再発明は避け、標準機能や信頼できる既存ライブラリで解決できる場合はそれを優先してください。
- 明確な理由がない限り、冗長な条件分岐や不要な防御コードは追加しないでください。

### ドキュメント管理

- API や画面仕様、運用方法に関わる変更を行った場合は、関連ドキュメントも必要に応じて更新してください。

## コマンド

- `npm run dev` - 開発サーバーを起動
- `npm run build` - 本番向けにビルド
- `npm run preview` - 本番ビルドをローカルでプレビュー
